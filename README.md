# CaFloMa
*The Cash Flow Manager   
Keeping track of your income and expenses*

---

### Adding new stuff to this project

The following rules should be applied if there is anything new going into this project.

#### Rule 1: Creating A Pull-Request

Making sure that new stuff always finds its way in main by merging a pull-request.
There should not be any commit directly going into main branch.

#### Rule 2: Use Semantic Commits

Making sure that commits always have the same structure. Therefore, the following semantic  
commit structured should be applied.

`type(scope): message`

##### type

|   type   |                 type description                |
|:--------:|:-----------------------------------------------:|
| feat     | new feature related                             |
| fix      | bug related                                     |
| docs     | documentation related                           |
| refactor | refactoring related                             |
| test     | testing related                                 |
| chore    | other changes that don't fit in the types above |

##### scope (optional)

Defines a scope where the changes made are being applied to.

##### message

Defines exactly in a short sentence what has been done.
